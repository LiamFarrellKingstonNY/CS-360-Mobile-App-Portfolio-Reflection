# CS-360-Mobile-App-Portfolio-Reflection
This project focused on developing a functional mobile app from the ground up, starting with UI design and ending with a working implementation. The main goal of my app was to create a simple and effective weight tracking tool that allows users to log their weight, view their progress over time, and stay consistent with their goals. The app was designed to solve a very practical problem—making it easy for someone to track progress without overcomplicating the process or requiring too much effort each time they use it.

To support those user needs, I designed the app around a few core screens and features. These included a main dashboard where users can quickly view their progress, a screen for logging weight entries, and simple navigation between sections. I intentionally kept the layout clean and straightforward so users wouldn’t feel overwhelmed. The UI design focused on making the most important actions—like entering weight or checking progress—easy to access right away. I think the design was successful because it removes unnecessary steps and keeps the experience focused on what the user actually wants to do.

When it came to coding the app, I approached it in smaller steps instead of trying to build everything at once. I focused on one feature at a time, made sure it worked, and then moved on to the next. This helped me stay organized and made debugging easier when something didn’t work as expected. I also paid more attention to writing code that I could understand later, which is something I didn’t always do before. This approach is something I will continue using in future projects because it makes the development process more manageable and less overwhelming.

To test the app, I used the Android Emulator to simulate how the app would run on a real device. I tested different features multiple times, such as entering data, navigating between screens, and checking that information was being displayed correctly. This process is important because it helps catch issues early and ensures the app actually works the way it’s supposed to. Testing also showed me where the app felt clunky or confusing, which helped me make improvements to both the functionality and the design.

One challenge I had to work through was making sure everything connected properly, especially when dealing with multiple screens and data updates. At times, things would work individually but not together, which forced me to rethink how different parts of the app interacted. I had to adjust my approach and be more intentional about how data was passed and updated across the app. This pushed me to think more like I was building a real product instead of just completing an assignment.

The area where I feel I was most successful was in creating a clean and user-focused interface. I was able to take what I learned about UI and UX and apply it in a way that made the app simple and easy to use. Instead of adding unnecessary features, I focused on making the core experience better, which I think made a big difference in the final result.

Overall, this project helped me understand the full process of developing a mobile app, from planning and design to coding and testing. It also changed how I approach development, I now think more about the user, build in smaller steps, and focus on creating something that is not just functional, but actually useful.
